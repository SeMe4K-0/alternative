# Comet Tracker - Расчет орбит комет

**Проект для хакатона 25-26 октября 2025**

## Описание проекта

Приложение для расчета орбиты кометы на основе астрономических наблюдений и предсказания минимального сближения с Землей. Проект вдохновлен фильмом "Don't Look Up" - когда обнаружена комета, необходимо точно рассчитать дату и дистанцию ее максимального сближения с Землей.

### Основная задача

Ввод от 5 и более астрономических наблюдений (прямое восхождение, склонение, время) → расчет 6 параметров орбиты (большая полуось, эксцентриситет, 3 угла, время прохождения перигелия) → определение даты и расстояния минимального сближения кометы с Землей.

## Использованные технологии

### Backend (Go)
- **Go 1.25** - основной серверный язык
- **GORM** - ORM для работы с базой данных
- **PostgreSQL** - реляционная БД для хранения данных
- **MinIO** - S3-совместимое хранилище для изображений наблюдений
- **Redis** - кеширование и сессии
- **Swagger** - авто-документация APIL
- **JWT** - аутентификация пользователей
- **Gorilla Mux** - роутинг
- **Zerolog** - логирование

### Python Microservice (Orbital Calculations)
- **FastAPI** - REST API для астрономических вычислений
- **Astropy** - работа с эфемеридами JPL (DE440, DE432s)
- **NumPy** - численные вычисления
- **SciPy** - оптимизация (minimize_scalar для поиска ближайшего приближения)
- **Custom time conversion** - преобразование RFC3339 ↔ Юлианская дата
- Вычисления по алгоритмам Кеплера для орбитальной механики

### Frontend (React + TypeScript)
- **React 19** - UI фреймворк
- **TypeScript** - типизация
- **Vite** - сборщик и dev-сервер
- **Tailwind CSS** - стилизация
- **Zustand** - управление состоянием
- **React Router** - маршрутизация
- **React Konva** - графический редактор для аннотации изображений
- **Axios** - HTTP запросы
- **date-fns** - работа с датами

## Архитектура проекта

```
hakaton/
├── backend/          # Go backend (API, auth, storage)
├── python/           # Python microservice (orbit calculations)
├── front/            # React frontend
└── convertation/      # Time conversion utilities
```

**Взаимодействие сервисов:**
1. Frontend → Go Backend (API запросы для CRUD операций)
2. Go Backend → Python Service (HTTP запросы для вычислений орбит)
3. Python использует JPL ephemerides через Astropy для точных позиций Комет/Марса


## Ключевые возможности

- ✅ Ввод астрономических наблюдений (RA, Dec, время) с фотографиями
- ✅ Загрузка и аннотация изображений через графический редактор
- ✅ Расчет 6 параметров орбиты (a, e, i, Ω, ω, Tp)
- ✅ Предсказание даты и расстояния минимального сближения с Землей
- ✅ Использование JPL ephemerides для точных позиций планет
- ✅ Аутентификация и личные кабинеты пользователей
- ✅ История всех расчетов
- ✅ Swagger документация API
- ✅ Unit-тесты для проверки корректности вычислений

## Проверка расчетов

Для валидации алгоритмов используются тестовые данные Марса (сравнение с известными орбитальными параметрами) и сверка с результатами JPL Horizons.
